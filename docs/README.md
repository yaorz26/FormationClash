# docs

项目需求文档与设计资料目录。

## 目录结构

```
docs/
├── README.md
├── re.md                          # 杂项记录
├── baseRequire/                   # 基础需求（原始设计）
│   ├── require.md                 # 核心规则、关键词与角色/职业说明
│   ├── require2.md                # 界面与后续优化需求
│   └── require3.md                # 攻击、受伤、死亡、依次结算等关键流程
├── newRequire/                    # 新增需求（扩展与维护）
│   ├── newCharacter.md            # 新角色：终焉、太阳守卫、帝法者
│   ├── newCharacter2.md           # 新角色：影忍者、小鲲、老王八、苍翾鹤
│   ├── newCharacter3.md           # 新角色：无畏、狂暴炸弹师、重力/虚弱/激怒、章鱼等
│   ├── bug.md                     # 当前或最近的修复项
│   ├── tester.md                  # 测试/修改器能力需求
│   ├── development_process.md     # 阶段化开发记录
│   └── animation.md               # 动画相关需求
└── docsForAi/                     # AI 接手资料
    └── ai_handoff.md              # AI 交接文档：项目概览、代码结构、规则约定、机制索引
```

## 文件说明

### baseRequire/ — 基础需求

游戏最初的核心设计文档，定义了基本规则体系。

| 文件 | 内容 |
|------|------|
| `require.md` | 游戏概括、角色/职业/阵营定义、选角/排阵/对战流程、阵型与移动规则 |
| `require2.md` | 界面布局需求、百科大全设计、后续优化方向 |
| `require3.md` | 攻击结算流程、受伤结算、死亡结算、反伤与抵御等关键战斗规则 |

### newRequire/ — 新增需求

持续迭代过程中产生的新角色、新机制和问题修复记录。

| 文件 | 内容 |
|------|------|
| `newCharacter.md` | 帝法者职业、终焉（复活）、太阳守卫（护盾）、护盾机制 |
| `newCharacter2.md` | 隐匿机制、影忍者、小鲲（位移）、老王八（全体嘲讽）、苍翾鹤（沉默光环） |
| `newCharacter3.md` | 无畏、狂暴炸弹师、重力/虚弱/激怒、章鱼/触手召唤等 |
| `bug.md` | 当前待修复或最近已修复的问题列表 |
| `tester.md` | 战斗修改器功能需求 |
| `development_process.md` | 各阶段开发内容记录 |
| `animation.md` | 战斗动画表现需求 |

### docsForAi/ — AI 接手资料

面向 AI 助手的项目上下文文档。

| 文件 | 内容 |
|------|------|
| `ai_handoff.md` | 项目概览、运行与测试命令、代码结构总览、数据与文档约定、关键战斗规则、已实现机制索引、UI 约定、修改器说明 |
